# Read-WeekPlan — инструкция

Инструмент показывает по линии (`L9RCDB` в `Data_boxprintingbin3v7.xlsb`) **выработку за текущую смену**
против **планового задания на смену**, плюс недельный итог. **SKU определяется автоматически** —
по тому, что реально идёт на линии сейчас. Вывод — на нидерландском.

---

## Что нужно

- **Windows** + установленный **Microsoft Excel** (файлы читаются через Excel COM).
- В папке со скриптом — `Read-WeekPlan.ps1`, `Read-WeekPlan.cmd` (консоль) и
  `Read-WeekPlan-Web.cmd` (браузер).
- **Пути к данным задаются в `config.txt`** (в той же папке). При первом запуске, если
  `config.txt` нет, скрипт **создаёт его сам** с дефолтными путями — открой и поправь под себя.

`config.txt`:
```
# Путь к файлу сбора коробок (фактическая продукция):
BoxPrintingFile = C:\...\Data_boxprintingbin3v7.xlsb

# Папка с файлами недельного плана (daily shift NDwkNN-YYYY.xls):
PlanFolder = C:\...\план

# Лист линии в файле сбора коробок (L9RCDB, L6RCDB, ...):
RcdbSheet = L9RCDB
```
Приоритет: **параметр командной строки > `config.txt` > встроенный дефолт**.

## Быстрый старт

Два способа запуска:

- **`Read-WeekPlan.cmd`** (двойной клик) — отчёт в **консоли** (монитор).
- **`Read-WeekPlan-Web.cmd`** (двойной клик) — **дашборд в браузере**: скрипт поднимает
  локальный веб-сервер, сам открывает браузер, страница авто-обновляется.

В обоих режимах:
1. сам определяет, **какой SKU сейчас идёт на линии**, и показывает отчёт;
2. раз в минуту проверяет **дату сохранения** `Data_boxprintingbin3v7.xlsb`;
3. как только файл пересохранён — перезагружает данные.

Остановить — **Ctrl+C** (в открывшемся окне).

### Веб-режим (дашборд)

`Read-WeekPlan-Web.cmd` или `... Read-WeekPlan.ps1 -Web`:
- поднимает локальный HTTP-сервер на `http://127.0.0.1:8770/` (порт меняется `-Port 9000`);
- автоматически открывает браузер (отключить — `-NoBrowser`);
- та же информация, что в консоли (таблица смены + недельные карточки), на тёмном дашборде;
- страница сама перезагружается каждые `-IntervalSeconds` секунд (по умолчанию 60).
- Использует только встроенный .NET (`TcpListener`) — **ничего ставить/качать не нужно**,
  права администратора не требуются.

## Как определяется SKU

1. **Автоматически** — в таблице смены показываются **все продукты, которые шли в текущей
   смене** (устойчиво к переналадке посреди смены). Тот, что **идёт сейчас** (последний час
   в `L9RCDB`), помечается `<< nu` и для него же выводится недельная сводка.
2. Если на линии **ещё ничего не сделано** (начало недели/запуск продукта) — берётся SKU,
   который **по плану** должен идти в этот день и смену.
3. Можно сфокусироваться на одном коде вручную: `-Sku 340049989`.

## Как читать отчёт

Пример с переналадкой посреди смены (два продукта в одной смене):
```
=========== LIJN L9RCDB - HUIDIGE PLOEG ===========
Tijd : 30/04/2026 20:30   ploeg 2 (13:00-21:00)   productiedag do 30/04
Week : 18 (26/04 - 03/05)   Plan: daily shift NDwk18-2026.xls
================================================
DEZE PLOEG (ploeg 2 (13:00-21:00)) - SKU : gemaakt / plan / nog te doen:
  340018283 LAYS 6 PACK 14X6X27.5G     571 /   759 /   188   75,2 %  << nu
  340064833 LAYS 6 PACK 14X6X27.5G     172 /   231 /    59   74,5 %
  ------------------------------------------------------------
  Totaal                               743 /   990 /   247

WEEK 18 - SKU die nu draait: 340018283 [actief op lijn]
  LAYS 6 PACK 14X6X27.5GR DK
  Weekplan      : 4.917 dozen
  Reeds gemaakt :   571 dozen   (11,6 %)
  Nog te maken  : 4.346 dozen   (88,4 %)
```

- **DEZE PLOEG** — строка на каждый продукт смены: `gemaakt` (сделано с начала смены) /
  `plan` (задание смены) / `nog te doen` (осталось) + % выполнения. `<< nu` = идёт сейчас.
  Если продукт один — строка одна, без `Totaal`.
- **WEEK** — недельная сводка по продукту, который **идёт сейчас**:
  `[actief op lijn]` / `[volgens plan]` / `[opgegeven (-Sku)]` — источник SKU.
- Все цифры — **на текущий момент**: учитываются часы только до «сейчас» (будущие смены = 0).
- **Производственный день стартует в 05:00**. Если сейчас, например, 02:00 — это ещё ночная смена
  предыдущего дня. Смены: 05:00–13:00, 13:00–21:00, 21:00–05:00.
- **Названия смен (вахты):** ночная (21:00–05:00) — **Z**; дневная (13:00–21:00) — **Y** при чётном
  номере недели, иначе **X**; утренняя (05:00–13:00) — наоборот (**X** при чётной, **Y** при нечётной).
  В отчёте смена подписывается буквой, напр. `ploeg Y (13:00-21:00)`. Номер недели берётся из плана.
- В плане **3 колонки на день = задания смен 1/2/3**. Сопоставление план↔факт — по дню месяца.

### Прогноз на конец смены (Prognose)

Блок **«PROGNOSE einde ploeg»** оценивает, сколько коробок будет сделано к концу смены при текущем
темпе. Темп считается **с момента запуска текущего SKU** (а не с начала смены):
- **Tempo** (коробок/мин) = сделано ÷ минут с момента запуска этого SKU;
- **Per uur** (коробок/час) = темп × 60;
- **Verwacht einde ploeg** = сделано + темп × минут **до конца смены**.
  Если SKU запустили в середине смены, до конца остаётся меньше 8 часов — это учитывается.

> «Текущее время» (и для темпа, и для остатка смены) берётся из **времени сохранения файла
> `Data_boxprintingbin3v7.xlsb`** (а не системных часов) — данные актуальны на момент сохранения.
> Момент запуска SKU = первый час, в котором этот SKU появился в смене. Если по SKU ещё ничего
> не сделано — прогноз не показывается.

## Режимы и параметры

```
powershell -ExecutionPolicy Bypass -File "Read-WeekPlan.ps1" [параметры]
```

| Параметр | Назначение |
|---|---|
| *(без параметров)* | консоль-монитор, авто-SKU, проверка раз в минуту |
| `-Web` | веб-дашборд в браузере (локальный сервер) |
| `-Port N` | порт веб-сервера (по умолчанию 8770) |
| `-NoBrowser` | не открывать браузер автоматически |
| `-Once` | разовый запуск в консоли, без мониторинга |
| `-IntervalSeconds N` | интервал проверки/авто-обновления (сек) |
| `-Sku CODE` | задать SKU вручную (отключает авто-выбор) |
| `-RcdbSheet NAME` | лист линии (по умолчанию `L9RCDB`) |
| `-Week NN` | конкретная неделя (иначе — последняя) |
| `-Now "ГГГГ-ММ-ДД ЧЧ:ММ"` | смоделировать момент (для проверки) |
| `-PlanFile "путь"` | явный путь к файлу плана |
| `-BoxPrintingFile "путь"` | явный путь к файлу сбора коробок |

## Настройка под «боевой» режим на заводе

Правится только **`config.txt`** (сам скрипт трогать не нужно):
- `BoxPrintingFile` — путь к **реальному** файлу сбора, обычно
  `F:\users\DATA_COLLECTIE\boxprinting\Data_boxprintingbin3v7.xlsb` (или сетевой путь);
- `PlanFolder` — папка, куда приходят недельные планы; на заводе это `\\…\Planning\FORECAST\`
  (файлы `daily shift NDwkNN-YYYY.xls`, лист «daily shift dpp»);
- `RcdbSheet` — линия (для большинства Lays-мультипаков `L9RCDB`; для других продуктов — их линия).

Любой из этих параметров можно временно переопределить из командной строки
(`-BoxPrintingFile`, `-PlanFile`, `-RcdbSheet`) — он перекрывает `config.txt`.

> ⚠️ Сам `.ps1` после ручной правки сохраняй в **UTF-8 с BOM** (Блокнот: «Сохранить как» →
> «UTF-8 с BOM»), иначе ломается кириллица/нидерландские символы. `config.txt` можно править
> обычным Блокнотом.

## Второй инструмент: Test-BoxCount (коробки по типам + прогноз)

`Test-BoxCount.ps1` — отдельный инструмент, который считает **не по агрегату `L9RCDB`, а по
сырым регистрациям коробок** на листе **`Boxruw9`** того же `Data_boxprintingbin3v7*.xlsb`
(строки с 11-й, колонка A = время, колонка B = ZPL-этикетка; **одна строка = одна коробка**,
тип продукта = тег `14`). План недели ему не нужен — только файл сбора коробок.

Запуск:
- **`Test-BoxCount-Web.cmd`** — дашборд в браузере (порт 8771);
- **`Test-BoxCount-Web-2010.cmd`** — то же, но «понарошку сейчас 21/07/2026 20:10» (порт 8772) —
  чтобы проверять на сохранённом снимке;
- `powershell -ExecutionPolicy Bypass -File Test-BoxCount.ps1 -Once` — разово в консоли.

Показывает за **текущую смену** (05–13 / 13–21 / 21–05): коробки по типам продукта,
итог, % от задания смены, график **коробок в минуту** с линией target
и **прогноз на конец смены**.

### Откуда берётся target на смену

Из **недельного плана** `daily shift NDwkNN-YYYY.xls` (лист `daily shift dpp`) — того же, что читает
Read-WeekPlan: колонка B = SAP-код, C = описание, дальше по **3 датированные колонки на день** =
задания смен 1 / 2 / 3.

- Файл ищется сам: свежий `daily shift NDwk*.xls*` в `PlanFolder` из `config.txt`
  (можно задать явно — `-PlanFile "путь"` или ключ `PlanFile`). Перебираются до 3 файлов,
  берётся тот, в чью неделю попадает производственный день.
- **Target смены = сумма плана тех продуктов, которые в эту смену реально шли** — план общезаводской,
  поэтому фильтруется по тому, что есть в `Boxruw9`. В таблице теперь колонки
  `Dozen nu / Plan ploeg / Nog te doen` по каждому продукту.
- **Если в смене ещё ничего не сделано** — берётся SKU с наибольшим планом на этот день+смену
  среди продуктов, встречающихся в листе (то есть «что должно идти по плану»).
- **Если дня нет в плане** (старый файл, как сейчас) или файла нет — откат на `ShiftTarget`
  из `config.txt` / `-ShiftTarget` (1234), сверху жёлтым выводится причина:
  `productiedag 21/07 (ploeg 2) staat niet in daily shift NDwk29-2026.xls (week 29, 12/07 - 19/07)`.
- Приоритет: **`-ShiftTarget` явным параметром > план > `config.txt` > 1234**.
  `-NoPlan` — вообще не открывать план (быстрее, target из config).
- Источник всегда подписан: строка `Target bron : ...` в консоли и мелким шрифтом под карточкой
  `Target ploeg` в дашборде.

Проверено на старом плане недели 29: 13/07 смена 1, продукт `340056955` → план **990** коробок
(в файле по 13/07 стоит `990 / 1056 / 1056` на три смены); сделано 602 → 60,8 %, прогноз 931 (94 %).

### Прогноз (PROGNOSE EINDE PLOEG)

- **Основной продукт = тот, что в ПОСЛЕДНЕЙ строке** (последняя зарегистрированная коробка),
  а не тот, у которого больше всего коробок. В таблице он помечен `nu` / `<< nu`.
- **Темп** = коробки текущего **непрерывного блока (run)** этого продукта ÷ минуты с первой
  коробки этого блока. Переналадка в середине смены обнуляет отсчёт — считается только то,
  что реально идёт сейчас.
- **«Сейчас» = min(часы, время сохранения файла)** и не раньше последней коробки: данные не
  «свежее» самого файла. Остаток = до конца смены.
- **Verwacht einde ploeg** = сделано (всего по смене) + темп × минут до конца смены.
- Вторая оценка — по темпу **последних `-RecentMinutes`** минут (по умолчанию 30), чтобы видеть
  разброс: если линия только что встала/ускорилась, эти два числа разойдутся.
- Ещё: **Nodig voor target** (какой темп нужен, чтобы дотянуть до задания) и **Target bereikt om**
  (во сколько будет достигнут target при текущем темпе; `na ploegeinde` = не успеваем).
- **Cumulatieve grafiek**: сплошная линия — фактический накопленный итог, пунктир — прогноз до
  конца смены, наклонная жёлтая — ровный темп под target. Затенённая часть справа = будущее.
- Если run слишком короткий (< 5 мин или < 5 коробок) — прогноз помечается «voorlopig».

### Простои (Stilstand) и отставание (Achterstand)

**Простой** = разрыв между соседними коробками больше `-StopMinutes` (по умолчанию 2 мин).
Считаются также разрыв «начало смены → первая коробка» (`opstart`) и «последняя коробка → сейчас»
(`staat nu stil` — если линия стоит прямо сейчас, это видно первым делом, красной плашкой).

- **Полоса `Stilstand`** — вся смена одной строкой: зелёное = линия шла, красное = стоп,
  оранжевое = стоит сейчас, серое справа = ещё не наступило. Стопы от 10 мин подписаны длительностью.
- **Карточки:** сколько всего минут простоя и в скольких стопах (+ самый длинный и когда),
  **Draaitijd** (% времени, когда линия реально шла), **Netto tempo** (коробок/мин **только на ходу** —
  это паспортная скорость линии, у вас ≈2,87/мин ≈ такт 21 с) и **Verlies stilstand** —
  сколько коробок стоило это время при нетто-темпе.
- **График `Achterstand`** — одна линия: на сколько коробок вы сейчас **впереди (+) или позади (−)**
  ровного темпа target. Ноль (жёлтый пунктир) = идём точно по заданию. Зелёная = впереди,
  красная = отстаём; пунктир вправо — куда придём к концу смены по прогнозу.
- Ниже — таблица самых длинных простоев с временем начала/конца.

Пример со смены 21/07 13:00–21:00: 62 мин простоя в 8 стопах, из них один на 39,5 мин (17:02–17:42);
Draaitijd 85,3 %; нетто-темп 2,87/мин; потеряно ≈178 коробок — ровно поэтому прогноз 1177 вместо 1234.

Параметры: `-Web`, `-Port N`, `-NoBrowser`, `-Once`, `-IntervalSeconds N`, `-Now "ГГГГ-ММ-ДД ЧЧ:ММ"`,
`-BoxPrintingFile "путь"`, `-BoxSheet Boxruw9`, `-PlanFile "путь"`, `-NoPlan`, `-ShiftTarget 1234`,
`-RecentMinutes 30`, `-StopMinutes 2`. Ключи `BoxSheet`, `PlanFile`, `PlanFolder`, `ShiftTarget`,
`RecentMinutes`, `StopMinutes` можно задать и в `config.txt`.

> Мелочь, о которой стоит знать: **первая строка (11-я) — не коробка**, а «значение на начало
> окна» дельта-выгрузки (всегда ровно на круглый час, обычно 05:00, и часто ещё от продукта
> предыдущей смены). Скрипт её пропускает — иначе в утреннюю смену попадала бы лишняя коробка
> чужого продукта.

> С `-Now` инструмент работает как машина времени: строки **позже** указанного момента
> не учитываются, поэтому картинка «как было в 15:00» честная.

## Если что-то не так

- **`Geen SKU…`** — на линии ничего не сделано И в плане нет задания на этот день/смену.
- **`Gemaakt = 0` в текущей смене** — смена только началась, либо этот продукт сейчас не идёт.
- **Неверная линия** — если факт всегда 0/неправдоподобен, проверь `-RcdbSheet` (линия продукта).
- **`... staat niet in het weekplan`** — на линии идёт продукт, которого нет в плане недели
  (выработку смены покажет, недельную сводку — нет).
- **Кракозябры** — `.ps1` сохранён без BOM (см. выше).
